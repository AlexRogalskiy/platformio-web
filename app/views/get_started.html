Getting Started

Learn how to install and use PlatformIO, configure and build your first cross-platform "Blink" project

Installation

PlatformIO is written in Python and works on Mac OS X, Linux, Windows OS and ARM-based credit-card sized computers (Raspberry Pi, BeagleBone, CubieBoard).

All commands below should be executed in Command-line application (Terminal):

  • Mac OS X / Linux – Terminal application
  • Windows – cmd.exe application.
  • Windows Users: Please Download the latest Python 2.7.x and install it. DON’T FORGET to select Add python.exe to Path feature on the "Customize" stage.

Paste each command at a Terminal prompt and run using pip (MAY require administrator access sudo):

Terminal

# update dependent packages to the latest versions
pip install -U pip setuptools

# install/upgrade the latest version of PlatformIO
pip install -U platformio

  • More installation options, troubleshooting and detailed information are located in Documentation.

Quickstart

  1. Find board type using Embedded Boards Explorer page or via platformio boards command
  2. Initialize new PlatformIO based project via platformio init command with the pre-configured environments for your boards:
    Terminal

    platformio init --board=TYPE_1 --board=TYPE_2 --board=TYPE_N

    Put your source code *.h, *.c, *.cpp, *.S, *.pde or *.ino files to src directory.

  3. Process the project's environments via platformio run command.

    Change working directory to the project's root where is located Project Configuration File (platformio.ini) and use these commands:

    Terminal
    # process/build project from the current directory
    platformio run

    # if you don't have enabled auto-uploading, then you can upload firmware manually
    platformio run --target upload

    # to clean cached/compiled files
    platformio run --target clean

"Blink" project example

With the help of PlatformIO we can build the same code for the different development platforms. Let's define example environments for the most popular boards from the different platforms:
Platform Board Framework
atmelavr Atmel AVR® 8- and 32-bit MCUs uno Arduino Uno (ATmega328P) arduino Arduino Wiring-based Framework
teensy Teensy teensy31 Teensy 3.1 arduino Arduino Wiring-based Framework
timsp430 MSP430 16-bit microcontrollers (MCUs) from TI lpmsp430g2553 MSP430G2553 LaunchPad energia Energia Wiring-based Framework
titiva TI TM4C12x MCUs ARM® Cortex®-M4 lptm4c1230c3pm Tiva C Series TM4C123G LaunchPad energia Energia Wiring-based Framework

The source code located below in main.cpp will turn ON and OFF the Wiring compatible board LED with intervals of 1 second (1000 milliseconds).

main.cpp 
#ifdef ENERGIA
  #include "Energia.h"
#else
  #include "Arduino.h"
#endif


#ifndef LED_PIN
  // Most Arduino boards already have a LED attached to pin 13 on the board itself
  #define LED_PIN 13
#endif

void setup()
{
  pinMode(LED_PIN, OUTPUT);     // set pin as output
}

void loop()
{
  digitalWrite(LED_PIN, HIGH);  // set the LED on
  delay(1000);                  // wait for a second
  digitalWrite(LED_PIN, LOW);   // set the LED off
  delay(1000);                  // wait for a second
}
platformio.ini 
; Project Configuration File
;
; A detailed documentation with the EXAMPLES is located here:
; http://docs.platformio.org/en/latest/projectconf.html

[env:uno]
platform = atmelavr
framework = arduino
board = uno

[env:teensy31]
platform = teensy
framework = arduino
board = teensy31

[env:lpmsp430g2553]
platform = timsp430
framework = energia
board = lpmsp430g2553

[env:lptm4c1230c3pm]
platform = titiva
framework = energia
board = lptm4c1230c3pm
build_flags = -DLED_PIN=GREEN_LED
  • The source code and results of this Blink Project can be downloaded from PlatformIO Project Examples repository.

Initialize empty project

PlatformIO-based project consists of 2 main parts:

  • Project Configuration File (platformio.ini) - where you can setup different environments with specific options (platform type, firmware uploading settings, pre-built framework, build flags and many more)
  • src directory - where your source code should be placed

There is platformio init command which allows you to initialize empty project. We recommend to run this command within empty directory.

Terminal

platformio init --board=uno --board=teensy31 --board=lpmsp430g2553 --board=lptm4c1230c3pm

Would you like to enable firmware auto-uploading when project is successfully built using `platformio run` command?
Don't forget that you can upload firmware manually using `platformio run --target upload` command. [y/N]:

The current working directory *** will be used for the new project.
You can specify another project directory via
`platformio init -d %PATH_TO_THE_PROJECT_DIR%` command.

The next files/directories will be created in ***
`platformio.ini` - Project Configuration File. |-> PLEASE EDIT ME <-|
`src` - Put your source code here
`lib` - Put here project specific (private) libraries
Do you want to continue? [y/N]: y
Project has been successfully initialized!
Useful commands:
`platformio run` - process/build project from the current directory
`platformio run --target upload` or `platformio run -t upload` - upload firmware to embedded board
`platformio run --target clean` - clean project (remove compiled files)

Firmware uploading

For uploading firmware to embedded device you can use platformio run command with --target upload option.

Terminal
platformio run --target upload

If you want to enable "auto-uploading" when source code is built successfully, then define for the specific environment's option targets = upload. More detailed information about targets option is available in the documentation.

  • Don't forget that you can process only specific environments -e or targets -t. Read more about it in platformio run command examples.
