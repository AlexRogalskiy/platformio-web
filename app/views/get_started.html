Installation

PlatformIO IDE for IoT development

PlatformIO IDE is the missing integrated development environment which provides comprehensive facilities for IoT development:
  • Cross-platform builder without external dependencies to the system software: 200+ embedded boards, 15+ development platforms, 10+ frameworks
  • C/C++ Intelligent code completion
  • C/C++ Smart code linter
  • Built-in Terminal with PlatformIO CLI (pio, platformio)

Try PlatformIO IDE now!

PlatformIO is written in Python (2.6 or 2.7 version is required) and works on Mac OS X, Linux, Windows OS and ARM-based credit-card sized computers (Raspberry Pi, BeagleBone, CubieBoard).

All commands below should be executed in Command-line application (Terminal):

  • Mac OS X / Linux – Terminal application
  • Windows – cmd.exe application.
  • Windows Users: Please Download the latest Python 2.7.x and install it. DON’T FORGET to select Add python.exe to Path feature on the "Customize" stage, otherwise pip command will not be available.

To install or upgrade PlatformIO paste command at a Terminal prompt and run using pip (MAY require administrator access sudo):

Terminal
pip install -U platformio

If pip command is not available, run easy_install pip or use PlatformIO Installer Script which will install pip and platformio automatically.

  • More installation options, troubleshooting and detailed information are located in the Installation section of documentation.

Quickstart

Step 1

Find board type using Embedded Boards Explorer page or via platformio boards command

Step 2
Initialize new PlatformIO based project via platformio init command with the pre-configured environments for your boards:
Terminal

platformio init --board=TYPE_1 --board=TYPE_2 --board=TYPE_N

# for example, initialize cross-platform project for Arduino Uno and Espressif Generic ESP8266 ESP-01
platformio init --board=uno --board=esp01

Put your source code *.h, *.c, *.cpp, *.S, *.ino, etc. files to src directory.

Step 3
Process the project environments via platformio run command.

Change working directory to the project's root where is located Project Configuration File (platformio.ini) and use these commands:

Terminal

# process/build project from the current directory
platformio run

# if you don't have enabled auto-uploading, then you can upload firmware manually
platformio run --target upload

# to clean cached/compiled files
platformio run --target clean

"Blink" project example

With the help of PlatformIO we can build the same code for the different development platforms. Let's define example environments for the most popular boards from the different platforms:
Platform Board Framework
Atmel AVR Arduino Uno (8-bit ATmega328P) Arduino Wiring-based Framework
Espressif NodeMCU 0.9 & 1.0 (32-bit ESP8266) Arduino Wiring-based Framework
Teensy Teensy 3.1 (32-bit ARM MK20DX256) Arduino Wiring-based Framework
TI MSP430 TI LaunchPad (16-bit MSP430G2553) Energia Wiring-based Framework
main.cpp 
/*
 * Blink
 * Turns on an LED on for one second,
 * then off for one second, repeatedly.
 */

#include "Arduino.h"

void setup()
{
  // initialize LED digital pin as an output.
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop()
{
  // turn the LED on (HIGH is the voltage level)
  digitalWrite(LED_BUILTIN, HIGH);
  // wait for a second
  delay(1000);
  // turn the LED off by making the voltage LOW
  digitalWrite(LED_BUILTIN, LOW);
   // wait for a second
  delay(1000);
}
platformio.ini 
; Project Configuration File
; Docs: http://docs.platformio.org/en/latest/projectconf.html


[env:uno]
platform = atmelavr
framework = arduino
board = uno

[env:nodemcu]
platform = espressif
framework = arduino
board = nodemcu
build_flags = -D LED_BUILTIN=BUILTIN_LED

[env:teensy31]
platform = teensy
framework = arduino
board = teensy31

[env:lpmsp430g2553]
platform = timsp430
framework = energia
board = lpmsp430g2553
build_flags = -D LED_BUILTIN=RED_LED
  • The source code and results of this Blink Project can be downloaded from PlatformIO Project Examples repository.

Initialize empty project

PlatformIO-based project consists of 2 main parts:

  • Project Configuration File (platformio.ini) - where you can setup different environments with specific options (platform type, firmware uploading settings, pre-built framework, build flags and many more)
  • src directory - where your source code should be placed

There is platformio init command which allows you to initialize empty project. We recommend to run this command within empty directory.

Terminal

platformio init --board=uno --board=nodemcu --board=teensy31 --board=lpmsp430g2553

The current working directory *** will be used for the new project.
You can specify another project directory via
`platformio init -d %PATH_TO_THE_PROJECT_DIR%` command.

The next files/directories will be created in ***
`platformio.ini` - Project Configuration File. |-> PLEASE EDIT ME <-|
`src` - Put your source code here
`lib` - Put here project specific (private) libraries
Do you want to continue? [y/N]: y
Project has been successfully initialized!
Useful commands:
`platformio run` - process/build project from the current directory
`platformio run --target upload` or `platformio run -t upload` - upload firmware to embedded board
`platformio run --target clean` - clean project (remove compiled files)

Firmware uploading

For uploading firmware to embedded device you can use platformio run command with --target upload option.

Terminal
platformio run --target upload

If you want to enable "auto-uploading" when source code is built successfully, then define for the specific environment's option targets = upload. More detailed information about targets option is available in the documentation.

  • Don't forget that you can process only specific environments -e or targets -t. Read more about it in platformio run command examples.
